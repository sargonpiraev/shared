The package ships one optional Agent Skill named **seodit**. It gives coding agents a short integration playbook for adding Playwright SEO checks to a Next.js app.

API details and examples stay in these docs; the skill only tells agents when to use seodit and which public guides to follow.

## Discovery metadata

The published `package.json` exposes docs and skill metadata:

```json
{
  "llms": "https://sargonpiraev.github.io/shared/seodit/llms.txt",
  "llmsFull": "https://sargonpiraev.github.io/shared/seodit/llms-full.txt",
  "skills": "https://github.com/sargonpiraev/shared/tree/main/packages/seodit/skills/seodit",
  "agents": {
    "skills": [{ "name": "seodit", "path": "./skills/seodit" }]
  }
}
```

- `llms` / `llmsFull` point agents to machine-readable documentation.
- `skills` is the git URL for `npx skills add`.
- `agents.skills` points to the skill inside the npm tarball for tools that discover skills from installed packages.

## Install the skill

### Option A - `npx skills`

```bash
npx skills add https://github.com/sargonpiraev/shared/tree/main/packages/seodit/skills/seodit \
  -a cursor \
  -y
```

Run once per project. Update later with:

```bash
npx skills update seodit
```

### Option B - `skills-npm`

```bash
npm i -D skills-npm
npx skills-npm setup
npm install @sargonpiraev/seodit
```

Use this option when you want the installed skill to match the `@sargonpiraev/seodit` version in your project.

## Agent workflow

Tell the agent what to integrate; referencing the skill is optional if the task clearly matches SEO / Playwright / seodit:

```text
Integrate @sargonpiraev/seodit in this Next.js app. Add Playwright SEO specs for /en and /ru.
```

The agent should:

1. Follow `SKILL.md` (if installed)
2. Run `npm install @sargonpiraev/seodit @playwright/test`
3. Read `llms` / `llmsFull` from the installed package `package.json`
4. Implement per [Getting started](/docs/getting-started) and [Playwright](/docs/playwright)
