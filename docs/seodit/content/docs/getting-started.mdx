## Installation

```bash
npm install @sargonpiraev/seodit @playwright/test
```

Install seodit in the app or test package that owns your Playwright configuration. It is a Playwright expect extension, so keep using your app's existing `@playwright/test` import.

### Agent skill (optional)

Use the bundled [Agent skill](/docs/agent-skill) so coding agents know how to integrate seodit.

**Option A — `npx skills` (git)**

```bash
npx skills add https://github.com/sargonpiraev/shared/tree/main/packages/seodit/skills/seodit \
  -a cursor \
  -y
```

Run this once in a project where you want your coding agent to have seodit-specific integration guidance.

**Option B — `skills-npm` (version-locked)**

```bash
npm i -D skills-npm
npx skills-npm setup
npm install @sargonpiraev/seodit
```

Use this option when you want the skill to follow the npm package version installed in your project.

## Extend expect

Import Playwright from your app, then extend it — do not import `test`/`expect` from the package.

```typescript
import { test, expect as baseExpect } from "@playwright/test";
import { extendSeoditExpect } from "@sargonpiraev/seodit/playwright";

const expect = extendSeoditExpect(baseExpect);

test("home page SEO", async ({ page }) => {
  await page.goto("/en");
  await expect(page).toHaveCanonical("https://example.com/en");
});
```

## Project layout

Split Playwright projects in `playwright.config.ts`:

- `app` → `**/*.test.ts`
- `seodit` → `**/*.seodit.spec.ts`

Co-locate specs next to routes, e.g. `src/app/[locale]/page.seodit.spec.ts`.

## Pure HTML assertions

For unit tests without a browser:

```typescript
import { assertCanonical } from "@sargonpiraev/seodit/html";

const issues = assertCanonical(html, "https://example.com/en");
```
